# Skill: design/aggregate-definition

## Purpose
Produce a complete Aggregate Definition for one aggregate вҖ” the most important DDD artefact. Defines the consistency boundary: invariants, state machine, commands handled, events emitted, and value objects. This is the specification a developer implements.

## Inputs
- `artifacts/design/domain/events.md`
- `artifacts/design/domain/commands.md`
- `artifacts/design/bounded-contexts.md`
- **Argument required:** aggregate name (e.g. `StorageLocation`, `FileProcessingJob`, `GoldenRecord`)

## Output
**File:** `artifacts/design/domain/aggregates/{name}.md` (kebab-case: `storage-location.md`)
**Registers in manifest:** yes

## Process
1. Read the event catalogue and bounded context map.
2. Identify all commands this aggregate handles and all events it emits.
3. Define invariants as boolean assertions that must always hold.
4. Define the state machine (valid states and transitions).
5. Identify value objects that belong to this aggregate.
6. Define identity strategy.
7. Run core/glossary вҶ’ validate. Write file. Register with manifest.

| Pending | Location registered but no valid credentials yet | Active |

## Commands Handled

### RegisterStorageLocation
| Attribute | Value |
|-----------|-------|
| **Description** | Register a new storage location with credentials |
| **Preconditions** | No location with the same path/platform already exists for this tenant |
| **Input** | `{ storage_path, platform, credentials, scan_configuration }` |
| **Validation** | Credentials must have read-only scope (verified against storage API) |
| **State transition** | None вҶ’ Pending |
| **Events emitted** | `StorageLocationRegistered` |
| **Rejects when** | Invariant 3 violated (write-scope credentials) |

### ValidateCredentials
| Attribute | Value |
|-----------|-------|
| **Description** | Validate stored credentials against the storage platform API |
| **Preconditions** | Location in Pending or ScanError state |
| **Events emitted (success)** | `CredentialsValidated` вҶ’ triggers state transition to Active |
| **Events emitted (failure)** | `CredentialValidationFailed` |

### InitiateScan
| Attribute | Value |
|-----------|-------|
| **Description** | Begin the initial full scan or an incremental scan |
| **Preconditions** | Location in Active state; no scan currently in progress (Invariant 2) |
| **State transition** | Active вҶ’ Scanning |
| **Events emitted** | `ScanInitiated` |

## Events Emitted

| Event | Trigger | Consumers |
|-------|---------|-----------|
| `StorageLocationRegistered` | RegisterStorageLocation command accepted | Audit Service, Worker Node Orchestrator |
| `CredentialsValidated` | Credentials confirmed read-only by storage API | (internal state transition trigger) |
| `ScanInitiated` | InitiateScan command accepted | Worker Node Orchestrator, Audit Service |
| `ScanCompleted` | Worker node reports scan completion | Graph Update Service, Audit Service |
| `ScanFailed` | Worker node reports unrecoverable error | Alert Service, Audit Service |

## Value Objects

### Credentials
```
Credentials {
  platform:      StoragePlatform (enum: GOOGLE_DRIVE | AWS_S3 | SHAREPOINT | DROPBOX)
  credential_type: CredentialType (enum: OAUTH2_SERVICE_ACCOUNT | API_KEY | IAM_ROLE)
  credential_ref:  string  // reference to secrets manager вҖ” never the credential itself
  scope_verified:  boolean
  verified_at:     Instant | null
}
```
*Immutable. To rotate credentials, replace the entire value object.*

### ScanConfiguration
```
ScanConfiguration {
  resource_cap_percent:  integer  // 0вҖ“100, max CPU% the scan may consume
  file_type_include:     string[] // empty = all types
  file_type_exclude:     string[]
  schedule:              CronExpression | null  // null = event-driven only
}
```

## Quality Checks
- [ ] Invariants are boolean assertions, not vague statements
- [ ] All commands have preconditions, validation, and events emitted defined
- [ ] State machine covers all possible states and transition paths including error states
- [ ] Value objects are immutable вҖ” no "update field" operations defined on them
- [ ] Credentials are stored as references to secrets manager, never inline
- [ ] No undefined ubiquitous language terms
